1) El programa original tendia mucho a repetir lineas de codigo y estas eran muy largas ademas de no tener una validacion para cuando se ingresaran 
letras en las calificaciones por lo tanto fallaba y se congelaba.
2) Decidimos crear metodos para el registro del alumno y las calificaciones para no saturar el proyecto con lineas innecesarias y si se necesita
volver a usarse ese fragmento del codigo solo se llame a la funcion.
3) Se utilizaron variable locales porque solo serian de utilidad en el metodo y no no eran necesarias para el resto del codigo.
4) La ventaja de los metodos es su reutilización ya que podemos volver a usar el fragmento sin tener que volver a teclear todo el codigo
dandonos un codigo más corto y más eficiente siendo este más facil de leer.
5) No es recomendable porque nos da más posibilidad de tener errores y a veces no las utilizamos todas provocando errores por variables sin uso.
6) Mejora ya que no se escribe muchas veces el mismo fragmento del codigo y se hace más facil identificar con estas que hace cada parte del codigo
ya que el codigo esta más reducido, claro y organizado y si estas estan identificadas de una manera especifica se ra más facil saber que hacen.
7) Se implemento una validacion para que cuando se ingrese una letra en las calificaciones el programa le indique al usuario que no puede
ingresar letras sino que solo números.
